The concept of grids

What is a grid?

In Grid Framework a grid is a mathematical lattice graph, an infinite regular tiling of the space it is embedded in. In plain terms this means we are sub-dividing the 3D space into an infinite series of regular cells.

Each grid is defined by a handful of basic properties specific to a certain type. The common properties are the position and rotation, which are directly taken from the Transform of the GameObject they are attached to. The origin of each grid lies where its position is.

When we look at a grid we see certain patterns: edges, vertices where edges cross, faces, which are areas enclosed by edges, and cells, which are spaces enclosed by faces.

Each grid forms a coordinate system that we can convert into and out of. The coordinate systems depend on the type of grid, some have only one and others have multiple. Rectangular grids have only one coordinate system (grid) while hexagonal grids have a total of five coordinate systems (herringbone up- and downwards, rhombic up- and downwards, and cubic). The details of every coordinate system are explained in the chapter about the respective grids.

Layered grids

Some grids are only two-dimensional in their nature, i.e. hexagonal and polar grids. In order to embed them in 3D space we add a third dimension by stacking layers of grids on top of each other. All layered grids inherit from a parent class.

Displaying grids

Since grids are infinitely large we need some way of displaying a finite subset of them. This task is delegated to special renderer objects that are attached to the same object as the grid. Depending on the grid there might be more than one way to display it, so there are different kinds of renderers and you can write your own renderer using the provided API.

When in editor mode grids are drawn using Unity's gizmos. At runtime when in scene mode Unity's low-level GL class is used instead. We have to attach a special component to the camera for rendering grids and the API allows you to write your own rendering implementation.
