Supporting other Unity plugins

Grid Framework can support other Unity plugins, but this support has to be enabled explicitly. Otherwise the code would not compile for people without those plugins and they would have to manually opt out of it.

At the moment Grid Framework supports the following plugins:

  • Vectrosity
  • Playmaker

Enabling support

To enable a feature you have to define a preprocessor symbol. Preprocessor symbols allow the code to be compiled if certain conditions are met. The syntax is the same for C# and UnityScript.

#define SOME_SYMBOL
// ... some time later
#if SOME_SYMBOL
// this code will be compiled
# else
// this code will not be compiled
#endif

There are many ways to define symbols in Unity that aren't limited to a particular script. The Unity manual lists all the possibilities, but you are most likely interested in enabling support globally for the entire project. To do so create a file called smcs.rsp (and us.rsp if you want to use UnityScript) in the root of your Assets directory. You can then list your symbols in the files like this:

-define:SOME_SYMBOL

Make sure you compiler is run after you define the symbols or else it won't get picked up. You have to define symbols in both files if you intend to use both C# and UnityScript.

Note
Make sure the files have their actual file extension if nothing happens. The Unity editor hides file extensions, so you should look at the files in a file browser instead.

List of symbols to define

Vectrosity

To enable Vectrosity in general define GRID_FRAMEWORK_VECTROSITY.

Playmaker

To enable the Playmaker actions define GRID_FRAMEWORK_PLAYMAKER.
